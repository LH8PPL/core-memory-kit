# v0.2.0 cut-gate — the clean-slate manual run before you tag

The final human gate before publishing v0.2.0 (D-24/D-25). The **wedge** (your
cross-project persona + project memory, portable, so a new project cold-opens
already knowing how you work) is the differentiator; this run confirms it *feels*
real on a fresh machine-state, and checks the three things automation can't.

**Already green (don't re-do by hand):** `npm run live-test` drives this whole
flow headlessly on the real tarball and passes every shipped-capability check
(install, the safe Task-69 skill, capture, persona-fill at `trust:high`, the
explicit `cmk` probes, cold-open). Its trail is in
[`../journey/live-test-runs/`](../journey/live-test-runs/). What you add by hand:
**R1** (console flash), **R2** (permission-prompt UX), and the **recall feel**.

The build vehicle (stages 0–3, "say it out loud" preferences, never "remember
this") is the same as [`v0.2.0-self-test-guide.md`](v0.2.0-self-test-guide.md) —
follow that for the build; this doc adds only the **cut-specific** checks.
