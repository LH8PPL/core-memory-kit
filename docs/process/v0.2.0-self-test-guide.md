# v0.2.0 self-test — step-by-step walkthrough (the v0.2.0 cut gate)

A literal, copy-along guide to test the kit by **building a small real thing across two sessions**, so memory gets exercised *organically* — **plus a full feature sweep** so nothing ships untested. This is the binding gate before cutting v0.2.0 (DECISION-LOG D-24): a fresh live test on a NEW project that exercises **everything the kit can do**. Based on the v0.1.1 guide; updated for the v0.2.0 fixes (F1 rich capture, F2 cross-project persona-fill, **the wedge's two halves — Task 78 auto-promote of stated rules at `trust: high` + Task 76 `cmk lessons promote`**) and the complete CLI/automatic surface.

The build vehicle follows [*Build Your Own Claude Code Web UI in 280 Lines of Python*](https://generativeai.pub/build-your-own-claude-code-web-ui-in-280-lines-of-python-7658422a8464) with **two deliberate twists**: backend = Claude Agent SDK (not Ollama), architecture = your layered FastAPI style (not single-file). The conflict is intentional — voicing your standards against the article's defaults is exactly the durable-preference signal the kit should capture.

The point isn't to *finish* the app — it's to do real, opinionated work in session 1 (stating decisions out loud, **never** "remember this") and prove Claude **recalls it in session 2** + the kit **learned how you work** (HABITS.md). Even stages 1–4 give a strong test. **The three checks that decide the cut: F1 (capture is rich), F2 (HABITS.md fills), recall (S2 reads it back).** The honest bar (D-26): if those don't make *you* say "whoa," that's signal — report what felt flat.

## SESSION 1 — install + build + capture

**a.** Fresh folder + open VS Code:
```powershell
mkdir C:\Temp\lior-test-5 ; cd C:\Temp\lior-test-5
git init
code C:\Temp\lior-test-5
```

**b.** In a VS Code terminal, scaffold (one command — `cmk install` sets up BOTH the project tier and the machine-global user tier):
```powershell
cmk install            # scaffolds context/ + the user tier (~/.claude-memory-kit/), wires hooks, appends CLAUDE.md
cmk doctor             # ✅ fresh install reads: 0 fail (skips for "nothing yet" / optional)
```
- [ ] **A1** `cmk install` → `scaffolded … files, hooks=wired`; surfaces the native-Auto-Memory coexistence heads-up; sets up the user tier too (`cmk init-user-tier` exists for an explicit user-tier-only re-init, but you do NOT need it after `cmk install`)
- [ ] **A3** `cmk doctor` → **`0 fail`, exit 0** (the fresh-install severity fix — must NOT show any fail). The pass/skip split varies: **with** memsearch installed ≈ `6 pass · 3 skip`; **without** it ≈ `4 pass · 5 skip` (HC-1 + HC-7 also skip — semantic search is the optional Layer 5b; keyword search still works). Every skip is "nothing's happened yet" (HC-3/4) or "optional" (HC-1/6/7). **The gate is zero failures, not a specific pass count.**

**c.** **No need to close VS Code** — Claude Code's file watcher picks up the hooks `cmk install` wrote. Type `/hooks` to confirm the 5 `cmk-*` hooks loaded. (`SessionStart` injection is tested in Session 2, a new session anyway.)

**d. Build it, stating preferences out loud.** Each stage has a **Build** prompt and a **Say it out loud** preference — a real engineering opinion, stated naturally, *never* "remember this." End the turn normally after each stage (the **Stop** hook fires auto-extract each time).

**Stage 0 — baseline (single file).**
- **Build:** *"Create a minimal Python web chat UI: a FastAPI server with a WebSocket endpoint and a single static index.html. Plain HTML/JS, no framework. Put the server in `app.py`."* → reply **"yes, run it"** when it offers to start the server.

**Stage 1 — set your architecture standards + refactor to layers** *(the big one — openly overrides the single-file shape)*.
- **Build:** *"Refactor this into a layered FastAPI project — `app/{api,services,repositories,schemas,core}/` + `app/main.py`. WebSocket route into `api/`, connection/broadcast logic into a service, Pydantic message schemas. Keep it on port 8000."*
- **Say it out loud:** *"How I build backends: FastAPI is the delivery layer, not the brain. Routes stay thin and just orchestrate; business logic lives in services; data access in boring repositories; Pydantic schemas are the boundary contracts. I want this even for a small app — I'd rather pay the structure cost now than fight it six months from now."*

**Stage 2 — swap the backend to Claude (the article's point).**
- **Build:** *"Change it from a broadcast room to a single-user chat with Claude via the Claude Agent SDK (`claude-agent-sdk`). Add a `ClaudeAgentService` wrapping `ClaudeSDKClient` with `include_partial_messages=True`. Each WebSocket connection gets its own session."*
- **Say it out loud:** *"Type hints on every function signature — I'm on Python 3.12+. Comments explain why, not what."*

**📍 Checkpoint A (after stage 2) — run the capture check, box f.** Your layered-architecture, Claude-Agent-SDK, and type-hint preferences should already be captured.

**Stage 3 — stream token-by-token.**
- **Build:** *"Stream Claude's output to the browser as it arrives — iterate `receive_response()`, handle `AssistantMessage`/`TextBlock` + `StreamEvent` deltas, push JSON frames over the WebSocket; client appends to the live bubble."*
- **Say it out loud:** *"Async all the way down — nothing blocking in the event loop. If something blocks, offload with `asyncio.to_thread`."*

**📍 Checkpoint B (after stage 3) — re-run box f.** The async/streaming preference should appear too.

**Stages 4–6 (optional, fuller experience).**
- **Stage 4 — sessions:** *"Persist + list sessions via a `repositories/` layer (JSON files); sidebar switcher. Repositories stay boring — read/write only."* → **Say:** *"Persistence is a repository concern. Storage details never leak into services or routes."*
- **Stage 5 — persona picker:** *"Replace the article's Ollama 'skills' with Claude Agent SDK system prompts — a sidebar picker that switches persona instantly, no model round-trip."* → **Say:** *"Configuration over conversation — if the UI can set state directly, don't make the user ask the model."*
- **Stage 6 — tooling:** *"Add slash-commands-as-buttons + a usage readout from `ResultMessage`."* → **Say:** *"Tooling I expect: `uv` for packages, `ruff` for lint+format, `pytest` for tests."*

> **⚠️ Session boundary.** Stages 0–6 all happen in **ONE** Claude Code session. Do **not** start a new session between stages. Build in Session 1 → new session → recall in Session 2.

**e. Explicit-capture checks (do these mid-build, in the same session terminal).**
- [ ] **C1 — terse:** `cmk remember "We deploy with Kamal to Hetzner, never Vercel."` → appears in `context\MEMORY.md`
- [ ] **C2 — RICH (F1):** `cmk remember "FastAPI is the delivery layer; logic in services, data in repositories" --type feedback --title "layered-backend" --why "pay the structure cost up front" --how "thin routes; push logic into app/services"` → creates a **rich fact file** `context\memory\feedback_layered-backend.md` with frontmatter + `**Why:**` + `**How to apply:**` (NOT a one-liner)
- [ ] **C3 — Poison_Guard:** `cmk remember "key sk-ant-api03-AAArealishlooking"` → **rejected** (exit 2), nothing written
- [ ] **C4 — sanitization:** `cmk remember "venv at C:\Users\<you>\proj\.venv\Scripts\python.exe"` → the file shows `~\…`, **never** your username

**f. Capture checkpoint — did it capture (auto)?** Re-run at Checkpoints A and B:
```powershell
cmk search "layered"
cmk search "type hints"
cmk search "port 8000"
type context\MEMORY.md
dir context\memory                 # the granular fact files
type context\memory\feedback_*.md  # ✅ F1: are the auto-captured durable facts RICH (Why/How)?
```
- [ ] **B1 — capture fires:** your decisions/preferences (Claude-Agent-SDK, layered FastAPI, thin routes, port 8000, type hints, async) show up — **without** ever saying "remember this"
- [ ] **B2 — F1 richness:** the durable preference facts are **rich fact files** (frontmatter + Why/How), not bare one-liners — this is the v0.1.2→v0.2.0 regression that was fixed
- [ ] **B3 — F2 the WEDGE:** `type %USERPROFILE%\.claude-memory-kit\HABITS.md` → it **FILLS** with your cross-project working style (layered backends, TDD, async) — sections **created if needed**. (Was empty pre-v0.2.0. This is wow #1.)
- [ ] **B4 — the AUTO half (Task 78): a STATED universal rule auto-promotes at `trust: high`.** Mid-build, in the chat (not a command), say **one rule with explicit universal phrasing**: *"From now on, in every project, always run `ruff` before committing — never skip it."* End the turn normally (the Stop hook fires auto-extract). Then:
  ```powershell
  type %USERPROFILE%\.claude-memory-kit\HABITS.md
  findstr /C:"trust: high" %USERPROFILE%\.claude-memory-kit\HABITS.md
  ```
  ✅ the rule appears in HABITS.md **on its own** (no command), and its provenance line reads **`trust: high`** + **`user-explicit`**. This is the Task-78 fix: an *explicitly stated* universal rule grades high → promotes durably (won't decay); a merely *observed* preference grades medium → parks in `~\.claude-memory-kit\queues\persona-review.md` instead. **Pre-Task-78, even a stated rule queued at medium and the persona never filled itself.** If B3 fills AND B4 shows `trust: high` on the stated rule, the wedge's auto half is real.

**g. ⬅️ START A NEW SESSION NOW.** You've built as many stages as you wanted, all in one session. Click the **new-chat / "+"** button (or reopen the folder) — that ends Session 1, starts Session 2, and fires the `SessionStart` injection hook.

- [ ] **D1 — recall:** Claude already knows the port, architecture, backend, and your style — **no re-briefing** (SessionStart injected the snapshot)
- [ ] **D2 — style follow-through (strongest signal):** it follows your engineering style on `/health` without you restating it

If it *doesn't* remember → that's the #1 thing to report (paste what it said + `context\MEMORY.md`).

- [ ] **D3 — stale-thread BASELINE (Task 68.1, watch for it):** look at `context\MEMORY.md` "Active Threads". Are there threads you already **finished** still listed as active?
