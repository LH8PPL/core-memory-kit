# v0.2.0 self-test ‚Äî step-by-step walkthrough (the v0.2.0 cut gate)

A literal, copy-along guide to test the kit by **building a small real thing across two sessions**, so memory gets exercised *organically* ‚Äî **plus a full feature sweep** so nothing ships untested. This is the binding gate before cutting v0.2.0 (DECISION-LOG D-24): a fresh live test on a NEW project that exercises **everything the kit can do**. Based on the v0.1.1 guide; updated for the v0.2.0 fixes (F1 rich capture, F2 cross-project persona-fill) and the complete CLI/automatic surface.

The build vehicle follows [*Build Your Own Claude Code Web UI in 280 Lines of Python*](https://generativeai.pub/build-your-own-claude-code-web-ui-in-280-lines-of-python-7658422a8464) with **two deliberate twists**: backend = Claude Agent SDK (not Ollama), architecture = your layered FastAPI style (not single-file). The conflict is intentional ‚Äî voicing your standards against the article's defaults is exactly the durable-preference signal the kit should capture.

The point isn't to *finish* the app ‚Äî it's to do real, opinionated work in session 1 (stating decisions out loud, **never** "remember this") and prove Claude **recalls it in session 2** + the kit **learned how you work** (HABITS.md). Even stages 1‚Äì4 give a strong test. **The three checks that decide the cut: F1 (capture is rich), F2 (HABITS.md fills), recall (S2 reads it back).** The honest bar (D-26): if those don't make *you* say "whoa," that's signal ‚Äî report what felt flat.

**d. Build it, stating preferences out loud.** Each stage has a **Build** prompt and a **Say it out loud** preference ‚Äî a real engineering opinion, stated naturally, *never* "remember this." End the turn normally after each stage (the **Stop** hook fires auto-extract each time).

**Stage 0 ‚Äî baseline (single file).**
- **Build:** *"Create a minimal Python web chat UI: a FastAPI server with a WebSocket endpoint and a single static index.html. Plain HTML/JS, no framework. Put the server in `app.py`."* ‚Üí reply **"yes, run it"** when it offers to start the server.

**Stage 1 ‚Äî set your architecture standards + refactor to layers** *(the big one ‚Äî openly overrides the single-file shape)*.
- **Build:** *"Refactor this into a layered FastAPI project ‚Äî `app/{api,services,repositories,schemas,core}/` + `app/main.py`. WebSocket route into `api/`, connection/broadcast logic into a service, Pydantic message schemas. Keep it on port 8000."*
- **Say it out loud:** *"How I build backends: FastAPI is the delivery layer, not the brain. Routes stay thin and just orchestrate; business logic lives in services; data access in boring repositories; Pydantic schemas are the boundary contracts. I want this even for a small app ‚Äî I'd rather pay the structure cost now than fight it six months from now."*

**Stage 2 ‚Äî swap the backend to Claude (the article's point).**
- **Build:** *"Change it from a broadcast room to a single-user chat with Claude via the Claude Agent SDK (`claude-agent-sdk`). Add a `ClaudeAgentService` wrapping `ClaudeSDKClient` with `include_partial_messages=True`. Each WebSocket connection gets its own session."*
- **Say it out loud:** *"Type hints on every function signature ‚Äî I'm on Python 3.12+. Comments explain why, not what."*

**üìç Checkpoint A (after stage 2) ‚Äî run the capture check, box f.** Your layered-architecture, Claude-Agent-SDK, and type-hint preferences should already be captured.

**Stage 3 ‚Äî stream token-by-token.**
- **Build:** *"Stream Claude's output to the browser as it arrives ‚Äî iterate `receive_response()`, handle `AssistantMessage`/`TextBlock` + `StreamEvent` deltas, push JSON frames over the WebSocket; client appends to the live bubble."*
- **Say it out loud:** *"Async all the way down ‚Äî nothing blocking in the event loop. If something blocks, offload with `asyncio.to_thread`."*

**üìç Checkpoint B (after stage 3) ‚Äî re-run box f.** The async/streaming preference should appear too.

**Stages 4‚Äì6 (optional, fuller experience).**
- **Stage 4 ‚Äî sessions:** *"Persist + list sessions via a `repositories/` layer (JSON files); sidebar switcher. Repositories stay boring ‚Äî read/write only."* ‚Üí **Say:** *"Persistence is a repository concern. Storage details never leak into services or routes."*
- **Stage 5 ‚Äî persona picker:** *"Replace the article's Ollama 'skills' with Claude Agent SDK system prompts ‚Äî a sidebar picker that switches persona instantly, no model round-trip."* ‚Üí **Say:** *"Configuration over conversation ‚Äî if the UI can set state directly, don't make the user ask the model."*
- **Stage 6 ‚Äî tooling:** *"Add slash-commands-as-buttons + a usage readout from `ResultMessage`."* ‚Üí **Say:** *"Tooling I expect: `uv` for packages, `ruff` for lint+format, `pytest` for tests."*

> **‚öÝÔ∏è Session boundary.** Stages 0‚Äì6 all happen in **ONE** Claude Code session. Do **not** start a new session between stages. Build in Session 1 ‚Üí new session ‚Üí recall in Session 2.
