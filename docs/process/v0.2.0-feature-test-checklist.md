# v0.2.0 fresh-live-test — complete feature checklist

The binding gate before cutting v0.2.0 (D-24): a fresh live test on a NEW project that exercises **everything the kit can do**, not just the F1/F2 fixes. Run from a fresh global install (`npm pack` the cli → install the tarball -g) in a brand-new project (e.g. `C:\Temp\lior-test-3`). Check each box; anything that fails is a v0.2.0 blocker.

Two test sessions: **S1** = build + capture; **S2** = a genuinely fresh Claude session that must *recall* what S1 captured (the part only a human can run honestly).

---

## A. Install & scaffold

- [ ] `npm install -g <tarball>` → `cmk version` prints `0.2.0`
- [ ] `cmk install` in the new project → scaffolds `context/` (MEMORY.md, memory/, sessions/, transcripts/, .locks/), `context.local/`, wires hooks into `.claude/settings.json`, appends the CLAUDE.md loader block
- [ ] Install surfaces the native-Auto-Memory coexistence heads-up
- [ ] `cmk init-user-tier` → scaffolds `~/.claude-memory-kit/` (USER.md, HABITS.md, LESSONS.md)
- [ ] `cmk doctor` on the fresh install reads **healthy** (`6 pass · 0 fail · 3 skip`, exit 0 — HC-3/4/6 skip, not fail)

## B. Automatic capture (the core — no commands, just work)

- [ ] **SessionStart inject**: open a session → the frozen snapshot (MEMORY.md + user tier) is injected into context (Claude "already knows" the loaded memory)
- [ ] **Stop-hook capture**: after a turn, `context/sessions/now.md` / transcripts get the turn (HC-4 transcripts firing)
- [ ] **Auto-extract**: state a durable preference in conversation ("I always use X / never Y") → after the turn, a fact lands in `context/memory/` or `MEMORY.md` **without** you saying "remember this"
- [ ] **F1 richness**: the auto-/explicit-captured durable fact is a **rich fact file** — frontmatter + `**Why:**` + `**How to apply:**`, not a bare one-liner
- [ ] **F4 check**: env facts from build work (framework, language version, port) land in Environment Notes (known weaker area — note if empty)

## C. Explicit capture

- [ ] `cmk remember "<one-liner>"` → terse bullet in MEMORY.md (sanitized, deduped)
- [ ] `cmk remember "<rule>" --why "<reason>" --how "<application>" --type feedback --title "<slug>"` → a **rich granular fact file** (F1)
- [ ] `--links a,b` → `related` cross-links in the fact frontmatter
- [ ] **Poison_Guard**: `cmk remember "key sk-ant-api03-…"` → rejected (exit 2), nothing written
- [ ] **Home-path sanitization**: `cmk remember "venv at C:\Users\<you>\…"` → the committed file shows `~\…`, never your username

## D. Recall & search

- [ ] `cmk search "<keyword>"` → finds the captured facts (FTS5 keyword)
- [ ] `cmk reindex` → rebuilds the index; search still works
- [ ] `cmk view` → inspect memory / audit log
- [ ] **S2 recall (the headline)**: open a FRESH session → Claude reads back the preference/decision S1 captured, without being re-told

## E. Cross-project persona — the wedge (wow #1)

- [ ] State cross-project doctrine in S1 ("I build every backend in layers", "always TDD") → after the turn, `~/.claude-memory-kit/HABITS.md` (or USER/LESSONS) **fills** — the section is created if needed (**F2**)
- [ ] `cmk persona generate` → manually runs the cross-project synthesis; candidates promote or land in `queues/persona-review.md`
- [ ] `cmk lessons promote <id>` → moves a project fact to the user tier
- [ ] **Cold-open proof**: open Claude in a DIFFERENT new project with the kit installed → the user-tier HABITS/USER inject, so it "knows how you work" before you say anything

## F. Lifecycle & self-curation

- [ ] `cmk daily-distill` → consolidates daily session summaries
- [ ] `cmk weekly-curate` → rolls `today-*.md` >7d into `archive.md`, dedupes
- [ ] `cmk compress <session>` / `cmk roll` → session compression / rotation
- [ ] `cmk register-crons` → registers the cron jobs with the host scheduler (HC-6); confirm the lazy-on-read fallback works when crons aren't registered

## G. Memory management

- [ ] `cmk forget <id>` → tombstones a fact (moved to archive/tombstones, still resolvable — not hard-deleted)
- [ ] `cmk purge --hard` → explicit hard delete (the only destructive path)
- [ ] `cmk trust <id> <level>` → change a fact's trust
- [ ] `cmk queue review` → walk pending medium-trust auto-extracts (promote/discard/skip)
- [ ] `cmk queue conflicts` → walk conflicts (keep-old/keep-new/merge-both/skip)

## H. Health, repair, config

- [ ] `cmk doctor` → 9 health checks (HC-1..HC-9) with accurate pass/fail/skip + repair commands
- [ ] `cmk repair --hooks` → re-wires hooks if settings.json drifts
- [ ] `cmk config get/set` (with `--show-origin`) → settings resolve across tiers

## I. Native coexistence & import

- [ ] `cmk disable-native-memory` → writes committable `autoMemoryEnabled:false`; `enable-native-memory` reverses; HC-8 reports the state
- [ ] `cmk import-anthropic-memory` → imports existing native Auto Memory files into the kit schema

## J. MCP server

- [ ] `cmk mcp serve` → stdio MCP server starts; `mk_remember` / `mk_get` / `mk_search` tools respond over JSON-RPC (reports `queued` state correctly, not false `accepted`)

## K. Transcripts

- [ ] `cmk transcripts extract <file>` → pulls durable facts out of a transcript

---

## Verdict

If **B (auto-capture + F1 richness)**, **E (HABITS fills + S2 cold-open)**, and **D (S2 recall)** all pass — that's the v0.2.0 bar (the model-of-a-car→car gap). Everything else green = ship. Any blocker → fix → re-test → then cut.

_The "wow" trio to feel, not just check: (B+C) rich capture, (D/E) fresh-session recall, (E) cross-project cold-open. If those don't make YOU say "whoa," that's signal, not just a failed checkbox (D-26)._
