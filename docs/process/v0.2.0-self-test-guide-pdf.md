# v0.2.0 self-test (B) ‚Äî the de-biased recall-quality run (PDF‚ÜíMarkdown scenario)

A second, **independent** copy-along self-test on a **deliberately different stack** from the FastAPI guide. Purpose: prove the recall-quality fixes (Task 84 compressor grounding, Task 80 setup/config capture, the domain-neutral guardrails) are **general**, not secretly fit to the one scenario that surfaced them.

> **Why a second guide (keep the [FastAPI one](v0.2.0-self-test-guide.md) as reference, do not delete it).** lior-test-5/6 reused the *same* scenario ‚Äî a layered FastAPI app on **port 8000**. That contaminates a recall test two ways (Lior 2026-06-03): **(1)** Claude's native Auto Memory may already hold that scenario from prior runs, and **(2)** a "pass" could just mean our guardrails happen to match FastAPI/8000 rather than being general. A brand-new stack neutralizes both ‚Äî the auto-memory has never seen Docling/CrewAI/Gemini, and our prompts were never written against them. This is the live form of Task 80b's "prove it on a different config." Build vehicle: [*How To Build Your Own Agentic PDF-to-Markdown Converter*](https://ai.gopubby.com/ai-pdf-to-markdown-converter-docling-crewai-python-7cdb2e2ffb31) (Docling + CrewAI + Gemini).

**What THIS run targets** (the FastAPI guide is the full feature sweep; this one is the recall-quality probe):

- **Task 84 ‚Äî the compressor stops hallucinating.** The new stack is dense hallucination bait: Haiku is tempted to write *CrewAI‚ÜíLangChain*, *Gemini‚ÜíGPT-4/Claude*, *Docling‚Üípdfplumber/PyPDF2*, or invent a *port* for a thing that has none. If the grounding rule holds, the snapshot states only what the session contained.
- **Task 80 ‚Äî setup/config capture beyond a port.** This project has **no port**. The recallable config is *which library / which model / which versions* ‚Äî a harder, more general test than "port 8000."
- **Task 84(b) ‚Äî stale-stage.** A mid-build pivot supersedes the Stage-0 file; the snapshot must not describe the dead file as current.
- **Domain-neutral guardrails (D-37).** Capture/recall must work without the prompts assuming "code"/"port"/"framework".

**User tier is WIPED clean** for this run (no lior-test-5/6 persona) ‚Äî so this is an honest capture-from-zero. The cross-project **cold-open** (wedge) needs a *populated* tier, so it's a stretch step at the end (needs a 2nd project after Session 1 fills the persona); it was already proven in lior-test-5/6, so it is **not** this run's gate.

> **Leave native Auto Memory ON (do NOT `cmk disable-native-memory`).** Most users run with Anthropic's native Auto Memory active by default, so a representative test runs the kit *alongside* it ‚Äî that's the realistic shape and it's what we want to validate (coexistence, both injecting at `SessionStart`). It does **not** re-contaminate this run: native memory is keyed per-project by folder slug, and `C:\Temp\lior-test-7` is a brand-new slug with an empty store ‚Äî zero old FastAPI/8000 facts to leak. (`cmk doctor` HC-8 will read "not active" until native memory writes its first file for this fresh folder ‚Äî that's `file_count: 0`, **not** disabled. It flips to "ACTIVE ‚Äî running ALONGSIDE the kit" once it writes, which is the state to expect.) Disabling it would make the test *less* like what a real user sees, not more.
