# v0.1.1 self-test — step-by-step walkthrough

A literal, copy-along guide to test the kit by **building a small real thing across two sessions**, so the memory features get exercised *organically*. The build vehicle is a mini "Claude Code-style web UI" in Python (inspired by *Build Your Own Claude Code Web UI in 280 Lines of Python*) — but the point isn't to finish the app, it's to do real work in session 1 and prove Claude **remembers it in session 2**.

**Prereqs:** global `@lh8ppl/claude-memory-kit@0.1.1` is installed (it is — `cmk --version` → `0.1.1`). Python 3 on PATH for the build. ~30–45 min per session.

---

## SESSION 1 — install + build + capture

**a.** Create a fresh folder:
```bash
mkdir c:\temp\lior-test-1
```
**b.** Open VS Code in it: `File → Open Folder… → c:\temp\lior-test-1` (or `code c:\temp\lior-test-1`).

**c.** Open a terminal in VS Code and install the kit into this project:
```bash
cmk install
```
Expect: `scaffolded … files, … hooks=wired`. Then run `cmk doctor` — HC-2 should PASS.

**d.** **Restart Claude Code** so the new hooks load: in the Claude Code panel type `/exit`, then start it again (or reopen the panel / run `claude`).

**e.** Now build, and **make real decisions out loud** as you go (this is what the kit should capture). Ask Claude Code things like:
- *"Create a minimal Python web chat UI: a FastAPI server with a WebSocket endpoint and a single static index.html. Use **FastAPI + uvicorn**, no frontend framework — plain HTML/JS. Put the server in `app.py`."*
- *"Stream responses token-by-token over the WebSocket."*
- State a couple of **preferences** plainly: *"From now on, keep your code comments minimal,"* and *"We're using port 8000 for this project."*
- Make a **decision**: *"We'll keep everything in a single `app.py` for now — no package split."*

Do ~20–30 min of real back-and-forth. End the turn normally (the Stop hook fires auto-extract).

**f. Checkpoint 1 — did it capture?** In the terminal:
```bash
cmk search "port 8000"
cmk search "FastAPI"
type context\MEMORY.md      # (Windows; or: cat context/MEMORY.md)
```
✅ **Expected:** your decisions/preferences (port 8000, single app.py, FastAPI, minimal comments) show up as facts in `context/MEMORY.md` — **without** you ever saying "remember this."

**g. Close VS Code completely.** (This is the real "new session later" condition.)

---

## SESSION 2 — the payoff (recall)

**a.** Reopen the **same** folder (`c:\temp\lior-test-1`) in VS Code, start Claude Code fresh.

**b.** Without re-explaining anything, ask something that needs session-1 context:
- *"What port are we using, and why is everything in one file?"*
- *"Add a /health endpoint to our server."* (does it use FastAPI + port 8000 + minimal comments, without being re-told?)

✅ **Expected (the whole product working):** Claude already knows the port, the single-file decision, the FastAPI choice, and your comment preference — because SessionStart injected the snapshot. **No re-briefing.** That's scenarios #5 + #6.

If it *doesn't* remember → that's the #1 thing to report (copy what it said + paste `context/MEMORY.md`).

## The lighter scenarios (any throwaway folder, ~10 min)

**Route B — `/plugin` install (first-ever end-to-end test).** In a *second* empty folder (`c:\temp\lior-test-2`), inside Claude Code:
```text
/plugin marketplace add LH8PPL/claude-memory-kit
/plugin install claude-memory-kit
/claude-memory-kit:bootstrap
/reload-plugins
```
✅ Expect: marketplace registers, plugin installs, bootstrap scaffolds `context/`, hooks active.
*(Faster local variant if the marketplace is slow to propagate: `claude --plugin-dir <path-to-the-cloned-repo>/plugin` loads it for one session.)*

**"Remember this"** — in any kit-installed session say: *"Remember this: the API base URL is https://api.example.com."* → check `cmk search "api.example.com"` finds it.

**Privacy tag** — say: *"`<private>` my license key is ABC-123 `</private>` — now add a README."* → check `context/MEMORY.md` does **NOT** contain `ABC-123`.
