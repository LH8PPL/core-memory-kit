# v0.1.0 requirements coverage audit

**Date**: 2026-05-29
**Auditor**: Claude (Opus 4.8), at Lior's request ("see if the kit really gives us all the abilities we talked about when we wrote requirements/design/tasks")
**Method**: extracted the full FR/NFR/US inventory from [`requirements.md`](../../specs/v0.1.0/requirements.md) + [`requirements-revisions-proposed.md`](../../archive/specs/v0.1.0/requirements-revisions-proposed.md), cross-referenced against [`tasks.md`](../../specs/v0.1.0/tasks.md) traceability footers and shipped code.

## Verdict

**Nothing specified for v0.1.0 was silently dropped.** Every requirement is either (a) shipped + tested + published, or (b) consciously deferred to v0.1.x/v0.2 with a tracking task or design §16 entry and a documented reason. The core promise — automatic cross-session memory for Claude Code — is delivered.

**One honest caveat**: "shipped" = code exists, tested, published, and the *engine* is proven (2026-05-29 live test: compression → auto-extract → snapshot → search all worked with real Haiku — see [`v0.1.0-live-test.md`](v0.1.0-live-test.md)). What is NOT yet confirmed is the **hooks firing automatically inside real Claude Code sessions** — that's Task 44 (real-usage test), best done after Task 49 makes the install shareable.

## User stories (US-1 … US-15)

All US map to shipped FRs except the search-scope nuance:

- US-1..US-8, US-11..US-13: ✅ covered by their FRs (continuity, overrides, auto+manual capture, private/retain, curation, fresh-clone, primary-cwd quirk).
- **US-9 (traceability via citation IDs)**: ✅ FR-14/15/29.
- **US-10 (fast search across ALL memory — incl. sessions + transcripts)**: ⚠️ **partial**. Keyword search over *durable* memory (MEMORY.md + granular archive) ships. Semantic (FR-18) deferred → Layer 5b. Indexing the *session-rollup/transcript* files themselves deferred → **Task 51** (v0.2, low priority — see note). *In practice most stated facts ARE searchable in v0.1.0 because auto-extract writes them to the indexed MEMORY.md; the gap is only content living solely in session rollups.*
- US-14/15 (in revisions-proposed): trust hierarchy + provenance → ✅ FR-29/FR-30, T7/T8.

## Anomalies investigated — both cleared

1. **FR-27 (viewer) "missing" from tasks** → confirmed *deliberately deferred* to v0.2 (tasks.md "What's deliberately not in tasks.md" line + design §16.4). Not an unmarked hole.
2. **`FR-52` / `FR-013` appearing in tasks.md** → these are **Cursor's** FR numbers from the four-spec experiment (our FRs stop at FR-30). They appear only inside descriptions referencing Cursor's external spec, deliberately backtick-wrapped so `validate-references.mjs` skips them (it would otherwise reject `FR-52` as undefined). Not dangling references to our own spec. Non-issue. (Documented at tasks.md 23.12.3 + D1-MIN-E.)

## Conclusion

The kit gives us the abilities we specified. v0.1.0 = 28/30 FRs + 9/9 NFRs shipped, with a clean, documented deferral ledger for the rest. The next concrete step toward "usable by someone else" is **Task 49** (unify install). The next toward "verified in the wild" is **Task 44** (real-usage live test, after 49). Auto-persona (Task 45) is the next net-new feature for v0.1.1.
