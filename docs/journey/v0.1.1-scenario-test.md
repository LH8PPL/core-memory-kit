# v0.1.1 scenario test — real user journeys

**Purpose**: validate the *published* `@lh8ppl/claude-memory-kit@0.1.1` by walking the journeys a real user takes — not a smoke test. This is Task 44 done as a scenario walkthrough, and it doubles as the script to hand a friend/tester.

**Why scenarios, not "does it install"**: the 2026-05-29 engine live test ([`v0.1.0-live-test.md`](v0.1.0-live-test.md)) proved the *engine* (compression → auto-extract → snapshot → search with real Haiku) but NOT (a) the hooks firing automatically inside a real Claude Code session, nor (b) the `/plugin` marketplace flow end-to-end. Those gaps are exactly what these scenarios cover.

**Who runs what:**
- 🤖 **CLI half (Claude can run from here)**: install in a sandbox, `cmk doctor` / `search` / `repair`, compression — against the published package.
- 🧑 **In-session half (needs a human in a real Claude Code session)**: hooks firing on Stop, SessionStart injection, the `/plugin` flow, the "remember this" skill. These require an interactive Claude Code session and can't be simulated.

Test on the **published** v0.1.1 (most faithful — catches packaging issues).

## CLI-half results — 2026-05-29 (🤖, against published `@lh8ppl/claude-memory-kit@0.1.1`)

All CLI-runnable scenarios **PASS**:

- **Hook bins on PATH** ✓ — all 5 (`cmk-inject-context`, `cmk-capture-prompt`, `cmk-observe-edit`, `cmk-capture-turn`, `cmk-compress-session`) resolve as bare names after `npm install -g`. This is the core cross-platform premise: Claude Code's shell-form hook commands will resolve.
- **#1 Route A install** ✓ — `cmk install` scaffolded 9 files, `.gitignore` + CLAUDE.md created, `hooks=wired`.
- **settings.json** ✓ — 5 events, bare-name commands, shell form (no `args`), **no `${CLAUDE_PLUGIN_ROOT}`**, correct matcher/async/timeouts.
- **#2 `cmk doctor`** ✓ — **HC-2 (hooks) PASS** (confirms the install→doctor composition fix live). HC-1/5/7/8/9 pass. HC-3/4/6 FAIL as *expected* on a fresh install (distill/transcripts/cron need a session or `cmk register-crons`).
- **#12 doctor→repair** ✓ — corrupted hooks → HC-2 FAIL → `cmk repair --hooks` → HC-2 PASS.
- **#13 provenance** ✓ — `dist.attestations` present, `predicateType: https://slsa.dev/provenance/v1`.

Tested in an isolated sandbox (`MEMORY_KIT_USER_DIR` redirected; not the repo). Global `@lh8ppl/claude-memory-kit@0.1.1` is installed on the dev machine (upgraded from 0.1.0) and ready for the in-session half.

## In-session half — awaiting human (🧑)

Needs a real Claude Code session (can't be simulated): **#3** Route B `/plugin` flow, **#4** "let Claude install it", **#5** auto-capture core loop, **#6** recall payoff, **#7** explicit "remember this", **#9** privacy tags. These are the highest-value gaps the engine live-test couldn't cover (hooks auto-firing in a session + the marketplace route).

## Verdict

CLI surface of the published v0.1.1 is **solid** — installs cleanly, wires correct cross-platform hooks, self-diagnoses + self-repairs, ships with provenance. Readiness to hand to a friend hinges on the **in-session half** passing (the hooks actually firing in a live session); run that next.
