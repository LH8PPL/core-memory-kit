## Thread 3 — Discovery: claude-mem already exists

**User**: *"i think we made something that already exist, tested and does it better, as much as i can see :( https://github.com/thedotmack/claude-mem"*

**Claude**: fetched repo metadata via `gh api`. Verified 77,244 stars, 6,656 forks, 30 releases, latest v13.3.0 published the same day (2026-05-21 10:26 UTC). Owned the miss explicitly: *"I should have checked first — same lesson you've flagged twice before. Logging that."* Saved a feedback memory in `youtube-to-slide/context/memory/feedback_check_existing_tools_first.md` (third strike on the pattern).

Performed a careful comparison: opaque global SQLite + Chroma vs our markdown per-project. Concluded they're **different design choices, not duplicates** — `claude-mem` is "OS-level memory the user has to manage globally"; ours is "project-level intent committed to git." Both are defensible.

**Outcome**: research note → [research/2026-05-21-claude-mem-architecture.md](../research/2026-05-21-claude-mem-architecture.md). Decision deferred to user.

## Thread 4 — "Can we fuse the systems?"

**User**: *"tell me the truth, can we make our project better if we take what they did? can we do a sort of fusion between solutions?"*

**Claude**: laid out a 4-phase roadmap (match hook coverage → AI compression → MCP+web viewer+FTS5 → cross-agent + multi-profile). Initial recommendation: **layer claude-memory-kit ON TOP of claude-mem** (claude-mem as OS-level memory, ours as project-level intent).

**User**: *"i want our own product, you can still steal how much you want from them, but i dont want to use claude-mem at all. so is everything you mentioned is the only things we are taking from them?"*

**Claude**: undersold the take list initially. Expanded honest list to 10+ capabilities (Setup hook + SessionEnd, citation IDs, `<retain>` tag, worker service, ~15 companion skills, E2E tests, CHANGELOG, IDE adapters, profile separation, MCP). Distinguished what to take vs explicitly skip (opaque SQLite, global scope, Chroma → use our memsearch).

**Outcome**: roadmap solidified. ADR-0002 written (markdown over opaque DB — keeping our choice with explicit reasoning).
