Multi-phase workflow for research-informed kit improvements:
1. **Comprehensive kit review** — study codebase, design, documentation, finished and open tasks, design weaknesses
2. **Thorough research material study** — review all source materials (papers, repositories, etc.) completely, not partial sections
3. **Systematic mapping** — map each research mechanism/finding to kit's actual architecture, documenting applicability and concrete reasoning
4. **Actionable outputs** — generate: ideas worth incorporating, refinements to existing tasks/lanes, new task candidates pinned to task numbers and release lanes

**Why:** Grounds research-informed work in product reality; ensures findings translate to concrete improvements rather than abstract exploration; maintains traceability to existing design/task structure

**How to apply:** When planning research-informed improvement cycles, apply this four-phase structure. Emphasize comprehensive input review and explicit mapping to existing kit architecture
