Scientific-method/epistemics lens on "how does memory EARN A>B oracle-free?" The transferable field is single-subject / N-of-1 causal inference — built precisely to earn a causal "A>B" for ONE subject from repeated within-subject observation, no benchmark. Decisive honest split (Daza 2018, PMC6087468): N1RT (randomized self-experiment) CAN earn a causal claim; N1OS (observational — the user tries things when they choose) is the WEAKEST form and the field explicitly scopes it to "discovering causal effects and possible N1RT treatments for SUBSEQUENT TESTING" — i.e. HYPOTHESIS GENERATION, not confirmed causation. A memory kit passively watching a user is doing N1OS. So the earned output is honestly a RANKED HYPOTHESIS ("A has out-performed B under conditions C, n=k, unconfirmed"), never a proven A>B. Core mechanics that transfer: (1) be-your-own-control / within-subject blocking (compare A-periods vs B-periods for the SAME user, differencing out their stable traits); (2) replication across multiple A/B periods (n>1 — a single success is the anecdotal/single-success trap + regression-to-mean); (3) "all else equal" attribution from ablation — vary ONE thing, hold context constant, else the win is confounded. Named failure modes the kit inherits wholesale: confounding (an unmeasured cause of BOTH the user's choice-of-method AND the outcome — fatal in observational data), non-stationarity/time-trend (the user/project/tooling drifts; A looked better only because it came later), carryover/washout (last method's effect bleeds into this period), the fundamental problem of causal inference (you never observe both counterfactuals for the same turn), the garden of forking paths (picking WHICH A>B comparison to record AFTER seeing the outcome inflates false "better-than"s even with no conscious fishing), and p-value peeking / optional-stopping (a kit that re-ranks every session and promotes A the moment accumulated evidence crosses a line inflates false positives — the sequential-testing trap). The oracle-free rescue that a single-user kit CAN adopt: anytime-valid inference / e-values (Ramdas) — evidence you may legitimately accumulate turn-by-turn and stop/peek at any time WITHOUT inflating error, the correct math for a memory that grows one episode at a time; plus specification-curve/robustness thinking for forking-paths. Kernel for cmk: model earned method-judgments as N1OS-grade ranked hypotheses with an attached n, a confound caveat, and a promotion GATE (accumulate as e-value-style evidence; only surface "A tends to beat B for you, unconfirmed, n=k" — never "A is better"). This LOWERS the bar honestly rather than faking an oracle.

**Why:** Answers the maintainer's open-question half ("how is better-than EARNED oracle-free, from OUTSIDE agent-memory?") from the scientific-method lens. The finding that flips the framing: epistemics HAS a field for single-subject oracle-free causal judgment (N-of-1), but its own literature draws the exact line the kit needs — observational single-subject (which is all a passive memory kit has) earns HYPOTHESES for later testing, not confirmed causal claims. So the honest ceiling for an oracle-free single-user kit is a ranked-hypothesis-with-caveat, not an earned A>B. This is the adversarial-honest answer the maintainer asked for: the lens does NOT need an oracle, but it caps the CLAIM STRENGTH at "provisional lead," and it names the precise confounds (non-stationarity, carryover, forking-paths, peeking) a naive "promote A after it worked" loop would fall into. Complements ADR-0017's prune>reinforce asymmetry: N1OS explains WHY reinforce-on-success is weak (silent success is unconfounded-unobservable) and why prune-on-failure is safer (a correction/red-test is a within-subject reversal signal).

**How to apply:** When designing cmk's method-judgment memory: (a) STORE method-judgments as N1OS hypotheses with schema {claim: "A>B", conditions, n_periods, confound_flags, status: provisional}; never store a bare "A is better." (b) Require n>1 before surfacing — one success is the single-success/regression-to-mean trap; the recurrence signal you already have IS the n-counter, but read it as within-subject replication not reinforcement. (c) Attach a stationarity/carryover caveat: if A and B were tried in different project eras/tooling, flag "time-confounded." (d) Use e-value / anytime-valid accumulation (not a fixed p<0.05 crossed once) so re-ranking every session doesn't inflate false better-thans — this is the correct sequential-testing math for a memory that grows one episode at a time. (e) Guard the garden-of-forking-paths: the kit, not a post-hoc glance, should PRE-COMMIT which method-pairs it tracks, else "A>B" gets manufactured by choosing the comparison after the outcome. (f) Prefer within-subject reversal evidence (user tried A, reverted to B, stuck with B) — that A/B/A pattern is the strongest oracle-free signal a single user emits, mirroring the reversal design. Cross-link to ADR-0017's judge-as-adapter (N1RT = a host that CAN randomize/test = richer judge; N1OS = passive IDE session = weakest judge) and the prune>reinforce asymmetry.
