- **Intake First**: Build full list of sources before reading anything. Triage against whole set prevents ad-hoc omissions (D-374 lesson).
- **Triage Criteria**: Evaluate each item against open backlog, three corpus gaps, and design tasks (95/127/189). Categorize as "read deeply," "skim one question," or "log in SOURCES.md and skip."
- **Code-level Reading**: For projects, read code not README (D-153 discipline). Code often contradicts docs.
- **One Dated Note Per Source**: Write to `docs/research/` immediately upon completion (not batched). Add SOURCES.md row with verification status.
- **Findings → Task Annotations**: Research outcomes annotate task entries. Verdicts include numbers or re-open triggers.
- **Multi-agent Approach**: For 8–10+ items, use parallel readers (one per source) with synthesis after. Proven in ECC study and recall trio (tasks 178/176/149).

**Why:** Past research cycles (D-374, D-153) showed batch intake prevents missing items and code-level reading catches contradictions. The process balances depth and coverage.

**How to apply:** Use this workflow for all research passes. Employ parallel readers for large corpus. Ensure code-level verification before task annotation.
