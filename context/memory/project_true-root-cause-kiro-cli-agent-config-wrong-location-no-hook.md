Then `kiro-cli agent list` revealed it: the ACTIVE default agent is the BUILT-IN `* kiro_default (Built-in)`, and agents live in `~/.kiro/agents/` (GLOBAL) + `<project>/.kiro/agents/` (workspace). BUT the kit writes its config to `~/.aws/amazonq/cli-agents/q_cli_default.json` — the OLD Amazon-Q location. kiro-cli 2.8.1 NEVER LOADS IT. So q_cli_default.json (name 'q_cli_default') just sits ignored while the built-in 'kiro_default' runs with ZERO hooks.
