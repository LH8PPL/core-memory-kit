Hermes's core function is skill-library curation and consolidation, not memory self-improvement:
- Its "insights" are pure statistics (token/session counts, no LLM reasoning)
- Its real loop: archive stale skills + merge overlapping skills into umbrellas
- It does NOT autonomously improve stored memories or generate insights

This matters because the project had investigated "memory-improvement loops," and Hermes seemed relevant—but it's orthogonal. Hermes solves skill-library bloat, not memory self-improvement.

**Why:** Clarifies Hermes's scope; prevents confusion when Task 179 positions both memory improvement AND skill curation as separate but related features.

**How to apply:** Treat Hermes as a reference for skill-curation patterns (track which skill absorbed which), not as a template for memory self-improvement.
