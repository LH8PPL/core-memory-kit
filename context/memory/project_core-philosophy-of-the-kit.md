Memory should be "readable, diff-able, and auditable in any editor" — a local-first, git-native, human-inspectable system rather than opaque centralized governance.

**Why:** Transparency and auditability are core to kit's value proposition versus enterprise fleet systems like memclaw

**How to apply:** Maintain plain-text frontmatter, git-native storage, and full audit trails; reject features that hide state in databases or opaque LLM enrichment
